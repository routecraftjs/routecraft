## About

The community registry for [Routecraft](https://routecraft.dev) capabilities, agents, and skills. Browse, install, and contribute reusable building blocks for your Routecraft projects.

[Routecraft core](https://github.com/routecraftjs/routecraft) is maintained by [DevOptix](https://devoptix.nl). This registry is maintained by the community: anyone can submit via pull request.

## What's in here

- **Capabilities**: TypeScript pipelines exposable as MCP tools, HTTP endpoints, CLI commands, or direct routes
- **Agents**: Capabilities with an LLM brain, defined persona, and explicit tool scope
- **Skills**: Markdown instruction files injected into agent context (compatible with OpenClaw skills)
- **Examples**: Starter project templates installable via `create-routecraft`

## Install

```bash
pnpm craft add elastic-logs
pnpm craft add elastic-logs@1.0.0
```

## How trust works

Every entry in this registry has its SHA-256 hash computed by CI, not by the author. When you run `craft add`, the CLI verifies the file you downloaded matches exactly what was present when that version was registered. If the source was modified after registration, the hash breaks and the install fails.

**SHA verified** means: the file is what it claims to be.
**SHA verified** does not mean: audited, safe for all use cases, or production-ready for your environment.

Review source code before deploying to production.

## Submit a capability

1. Add your files under `capabilities/<id>/<version>/`
2. Include a `craft.yml` manifest (no `sha256` field: CI adds it)
3. Open a pull request
4. CI validates schema, runs static analysis, and computes the SHA
5. A maintainer reviews and merges

Full submission guide: [routecraft.dev/docs/registry](https://routecraft.dev/docs/registry)

## The craft.yml manifest

```yaml
type: capability
id: elastic-logs
name: Elasticsearch Log Fetcher
version: 1.0.0
description: >
  Fetches logs from Elasticsearch for a given traceId.
author: routecraftjs
license: Apache-2.0
tags:
  - observability
  - elasticsearch
dependencies:
  "@routecraft/routecraft": "^0.4.0"
requiredCapabilities:
  - history-search@1.0.0
env:
  - ELASTIC_URL
  - ELASTIC_API_KEY
```

| Field | Required | Description |
|-------|----------|-------------|
| type | Yes | `capability`, `agent`, `skill`, or `example` |
| id | Yes | Lowercase, hyphen-separated unique identifier |
| version | Yes | Semantic version (immutable once merged) |
| description | Yes | What the capability does and when to use it |
| author | No | Author or organization name |
| license | No | SPDX license identifier |
| tags | No | Discovery tags |
| dependencies | No | npm packages to install (package name to version range) |
| requiredCapabilities | No | Other registry capabilities this one depends on |
| env | No | Environment variable names the capability requires |
| sha256 | No | Computed by CI. Never authored manually. |

Versions are immutable. Once a version is merged, its content cannot be changed. To ship a fix, publish a new version (e.g. `1.0.1`) in a separate directory.

## Repository structure

```
routecraft-registry/
  capabilities/       # TypeScript capability files
  agents/             # Agent capability files
  skills/             # Markdown skill files
  examples/           # Starter project templates
  registry/           # Generated by CI, do not edit manually
  scripts/            # Open source CI scripts, reusable in your own registry
```

## Run your own registry

The scripts in `scripts/` are Apache 2.0 and reusable. Set up a private registry with the same directory structure, then point `craft add` at it:

```bash
pnpm craft add my-internal-cap --registry https://registry.internal.acme.com --allow-unofficial
```
