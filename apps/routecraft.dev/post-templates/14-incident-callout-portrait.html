Yet another agent just nuked production.

An autonomous coding agent ran a "clean up disk space" task and dropped a 47 GB Postgres database. It had been given root SSH and a single instruction. It did exactly what it was told.

Via TechCrunch ↗
How this doesn't happen on Routecraft
cleanup.ts bounded capability 

craft()
  .id('cleanup-old-logs')
  .authorize({ roles: ['ops'] })
  .guard(onlyLogsOlderThan('30d'))
  .to(file({ action: 'delete' }))

The agent only sees what you handed it. No SSH. No root. No surprise DROP TABLE.
